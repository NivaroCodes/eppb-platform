# План разработки и распределение ролей (4 разработчика)

## 1. Команда и роли
*   **Backend Dev 1 (Engine):** Ядро системы, Workflow Engine, Rule Evaluator, логика переходов.
*   **Backend Dev 2 (API/Data):** REST API, Схема БД, Персистентность, Слой интеграции (Mock EISH).
*   **Frontend Dev 1 (Builder):** Админ-панель, Конструктор форм, Редактор бизнес-логики.
*   **Frontend Dev 2 (Portal):** Каталог услуг, Dynamic Renderer, Wizard UX, Dashboard.

---

## 2. Фазы разработки

### Phase 1: Core MVP (0-24ч) — Минимально рабочая система
**Цель:** Запуск простейшей услуги из JSON-схемы.

| Роль | Задачи | Приоритет | Зависимость |
| :--- | :--- | :---: | :--- |
| **BE 2** | Инициализация проекта, настройка БД, API эндпоинты `/services` | High | - |
| **BE 1** | Определение структуры JSON-схемы, базовый парсер схемы | High | BE 2 (DB) |
| **FE 2** | Dynamic Renderer: отрисовка input/select полей из JSON | High | BE 1 (Schema) |
| **FE 1** | CRUD интерфейс для управления услугами, базовый JSON-редактор | Medium | BE 2 (API) |

### Phase 2: Demo Ready (24-48ч) — Готовность к показу жюри
**Цель:** Реализация кейса "Лизинг" с ветвлением и валидацией.

| Роль | Задачи | Приоритет | Зависимость |
| :--- | :--- | :---: | :--- |
| **BE 1** | **Workflow Engine:** Логика переходов между шагами на основе условий | High | FE 2 |
| **BE 2** | **Integration Mock:** API для автозаполнения данных по БИН (eGov) | High | BE 1 |
| **FE 2** | **Wizard Controller:** Многошаговость, переходы `next/prev`, валидация | High | BE 1 |
| **FE 1** | **Visual Builder:** Drag-n-drop полей, настройка простых правил (if/else) | High | FE 2 |

### Phase 3: Advanced (48-72ч) — WOW-эффект
**Цель:** Дополнительные функции и "полировка".

| Роль | Задачи | Приоритет | Зависимость |
| :--- | :--- | :---: | :--- |
| **BE 1** | **Calculated Fields:** Вычисляемые значения в форме (сумма, налоги) | Medium | BE 1 |
| **BE 2** | **Audit Log & Security:** Логирование действий, RBAC | Medium | BE 2 |
| **FE 2** | **Status Dashboard:** Продвинутый трекинг статусов, уведомления | Medium | BE 2 |
| **FE 1** | **Preview Mode:** Режим предпросмотра формы в реальном времени | Low | FE 2 |

---

## 3. Риски и решения

*   **Риск:** Конфликты в структуре JSON-схемы между BE и FE.
    *   **Решение:** Зафиксировать контракт схемы в первый час хакатона. Использовать JSON Schema для валидации.
*   **Риск:** Сложность визуального редактора логики.
    *   **Решение:** Начинать с табличного/списочного представления правил, переходить к визуальным графам только если останется время.
*   **Риск:** Отставание одного из разработчиков.
    *   **Решение:** Передача части задач (например, верстки Dashboard) от FE 2 к FE 1.

## 4. Порядок выполнения (Roadmap)
1. **BE 2 + BE 1:** Описание контракта API и JSON-схемы (Час 1-2).
2. **BE 2:** Создание БД и базовых API (Час 2-6).
3. **FE 2 + BE 1:** Синхронизация рендерера и парсера схемы (Час 6-12).
4. **FE 1:** Запуск админки для загрузки схем (Час 12-18).
5. **Вся команда:** Интеграция сценария "Лизинг" (Час 24-36).
6. **Полировка и подготовка презентации:** (Час 60-72).
